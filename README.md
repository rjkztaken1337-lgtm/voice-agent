# voice-agent

Голосовой ассистент с активацией по имени ("Привет, Влад") и клонированным голосом (Coqui XTTS v2). Локальные STT (mlx-whisper) и VAD (webrtcvad) для непрерывного прослушивания без кнопки. Мозг агента — Claude Code CLI в headless-режиме, работающий через существующую подписку Claude Code (OAuth), без отдельного платного Anthropic API-ключа.

## Установка

1. Python 3.11 через Homebrew (`brew install python@3.11`), venv в `.venv/`.
2. `pip install -r requirements.txt`.
3. Положить чистую запись голоса друга (10–30 сек, без шума/музыки, WAV) в `voice_sample/friend.wav`.
4. Убедиться, что Claude Code уже авторизован (обычный вход в VS Code расширение/CLI) — отдельный `ANTHROPIC_API_KEY` не нужен.
5. При первом запуске macOS спросит разрешение на микрофон — разрешить.

## Запуск

```
source .venv/bin/activate
python main.py
```

Агент постоянно слушает микрофон через VAD. Скажи "Привет, Влад" (можно сразу с командой, например "Привет, Влад, который час") — ассистент ответит голосом друга.

## Как устроен "мозг"

`agent.py` вызывает нативный бинарник Claude Code CLI (`claude -p ...`) в JSON-режиме на каждый ход, с `--permission-mode bypassPermissions` (иначе диалоги подтверждения инструментов не с кем показывать — нет интерактивного терминала) и встроенными инструментами `Bash,Read,Write,WebSearch`. Память между перезапусками — через `--session-id`/`--resume` самого CLI (id сессии хранится в `db/cli_session_id.txt`), без отдельной SQLite-истории.

## Безопасность

Раз подтверждения инструментов на уровне ОС отключены (`bypassPermissions`), защита от опасных команд (rm, sudo, force-push и т.п.) реализована через системный промпт: агент обязан сначала словами описать, что собирается сделать, и дождаться голосового "да" в следующей фразе, прежде чем реально вызвать инструмент. Это программная, а не аппаратная защита — полагается на то, что модель следует инструкции.
